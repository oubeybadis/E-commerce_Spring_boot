# PROJET

# Enoncé :

Le projet vise à développer une application web type Ouedkniss.com. L'application devra aussi être accessible à travers une application mobile.

Avant d'entamer la conception et la mise en œuvre de la nouvelle plateforme, une étude des points forts et points faibles de l'application Ouedkniss.com devra être conduite. Cela permettra d'incorporer des améliorations dans la nouvelle plateforme qui sera mise en œuvre.

Dans le cadre de ce projet, il est demandé :

1) Etablier le diagramme des cas d'utilisation.  
2) Décrire chaque cas d'utilisation par un diagramme de séquence système.  
3) Etabir le diagramme de classes d'analyse pour ce système. Ajouter aux classes tous les attributs jugés importants.  
4) Proposer une architecture en couches pour ce système. Etabir le diagramme de paquetages correspondant.  
5) Décrire comment la solution proposée sera déployée à travers un diagramme de déploiement.  
6) Implémenter le système en utilisant le framework Spring Boot.

# Consignes :

- Pour la partie back-end, on utilisera Spring Modulith.  
- Pour la partie front-end web, on utilisera Thymeleaf.  
- Pour la partie accès aux données, on utilisera Spring Data et l'ORM Hibernate.  
- Pour le SGBD, on utilizes MySQL.  
- Un rapport contenant les différents diagrammes ainsi que les可以选择 effectuels devra être remis à la fin du projet. Le rapport ne devra désigner que ce qui a été réellement impliqué et ne devra pas dépasser 10 pages.  
- Le projet devra être réalisé en binôme.  
- Pour le barème, 6 points seront accordés à la partie analyse/conception, 11 points pour l'application web et 3 points pour l'application mobile. Le barème dépendra également du nombre de fonctionnalités implémentées et de leur importance. Les tests aussi seront pris en considération lors de la notation.
